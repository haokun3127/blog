已发布题目

下面按 LeetCode Hot100 官方题型顺序整理。每篇都尽量保留“推荐解法”和“最容易错的点”，这样回看时不需要重新翻整篇正文。

哈希

题目 推荐解法 最容易错的点
Hot100：哈希1-两数之和 一次遍历 + HashMap 先查再放，避免同一个元素用两次
Hot100：哈希2-字母异位词分组 排序字符串作为 key char[] 不能直接用 toString() 当 key
Hot100：哈希3-最长连续序列 HashSet 找起点，再向后扩展 对每个数都向后扩展，导致重复计算

双指针

题目 推荐解法 最容易错的点
Hot100：双指针1-移动零 两次遍历：先搬非零，再补零 忘记最后补 0，或者把非零元素顺序打乱
Hot100：双指针2-盛最多水的容器 左右双指针向中间收缩 不理解为什么移动短板，而不是移动长板
Hot100：双指针3-三数之和 枚举 i，在 i + 1 到末尾之间做两数之和 只顾着找答案，忘了给 i、L、R 做去重，导致结果重复
Hot100：双指针4-接雨水 左右双指针向中间收缩 不清楚为什么比较的是 leftMax 和 rightMax，而不是直接比较当前柱子高度

滑动窗口

题目 推荐解法 最容易错的点
Hot100：滑动窗口1-无重复字符的最长子串 可变窗口维护无重复子串 重复字符出现后不能只移动一次 left，而要用 while 一直缩到窗口合法。
Hot100：滑动窗口2-找到字符串中所有字母异位词 固定长度窗口 窗口必须保持固定长度；一旦超过 p.length() 就要立刻移除左端字符。

子串

题目 推荐解法 最容易错的点
Hot100：子串1-和为 K 的子数组 前缀和计数 map 里存的是“出现次数”，不是最后一次出现的位置。
Hot100：子串2-滑动窗口最大值 单调递减双端队列 队列里一定要存下标，这样才能判断元素是否已经滑出窗口。
Hot100：子串3-最小覆盖子串 可变窗口覆盖 判断窗口是否覆盖 t，看的是 count == 0，不是窗口长度。

普通数组

题目 推荐解法 最容易错的点
Hot100：普通数组1-最大子数组和 Kadane 算法 current 和 ans 都应该从 nums[0] 开始，不能从 0 开始，否则全负数组会出错。
Hot100：普通数组2-合并区间 排序后线性合并 不排序就直接合并，结论通常不可靠。
Hot100：普通数组3-轮转数组 三次反转 别忘了先做 k %= n，否则当 k 很大时前后边界会出错。
Hot100：普通数组4-除了自身以外数组的乘积 答案数组压缩辅助空间 这题不能用除法，所以要明确拆成前缀积和后缀积两部分。
Hot100：普通数组5-缺失的第一个正数 值放回对应下标 交换时一定要用 while，因为一次交换后当前位置可能还是不对。

矩阵

题目 推荐解法 最容易错的点
Hot100：矩阵1-矩阵置零 借第一行和第一列做标记 第一行和第一列既是数据区又是标记区，所以必须额外用两个布尔变量保存它们原本是否该清零。
Hot100：矩阵2-螺旋矩阵 按层模拟遍历 遍历完上边和右边后，处理下边和左边前要重新判断边界是否仍然有效。
Hot100：矩阵3-旋转图像 转置 + 每行反转 转置时内层循环必须从 j = i + 1 开始，否则会把元素交换两次。
Hot100：矩阵4-搜索二维矩阵 II 从右上角出发 起点最好固定在右上角或左下角，这样比较结果才能天然决定移动方向。

链表

题目 推荐解法 最容易错的点
Hot100：链表1-相交链表 双指针换头对齐长度 比较的是节点地址是否相同，不是节点值是否相同。
Hot100：链表2-反转链表 迭代反转 反转前一定要先保存 cur.next，否则后续链表会断掉。
Hot100：链表3-回文链表 找中点后比较前后两半 比较时以后半段链表是否走完为准，因为它长度不会超过前半段。
Hot100：链表4-环形链表 Floyd 判圈 判断的是节点是否相遇，不是节点值是否相等。
Hot100：链表5-环形链表 II Floyd 找入环点 只有第一阶段真的相遇了，才进入找入环点阶段。
Hot100：链表6-合并两个有序链表 虚拟头节点 + 双指针 用虚拟头节点可以避免“第一个节点怎么接”这种分支判断。
Hot100：链表7-两数相加 逐位相加 + 进位 两条链表长度可能不同，所以空位要按 0 处理。
Hot100：链表8-删除链表的倒数第 N 个结点 让快指针先走 n 步 使用 dummy 可以统一删除头节点的情况。
Hot100：链表9-两两交换链表中的节点 虚拟头节点 + 成对交换 交换的是节点本身，不是简单交换节点值。
Hot100：链表10-K 个一组翻转链表 找到一组再局部反转 每组反转前要先确认节点数是否达到 k 个。
Hot100：链表11-随机链表的复制 原节点到新节点的一一映射 深拷贝必须复制节点关系，不能让新链表继续引用旧链表节点。
Hot100：链表12-排序链表 分治 + 合并有序链表 找到中点后一定要把链表断开，否则递归会一直处理同一段链表。
Hot100：链表13-合并 K 个升序链表 优先队列维护当前最小节点 堆里放的是节点本身，不是单纯放节点值。
Hot100：链表14-LRU 缓存 O(1) 维护最近使用顺序 光有哈希表只能快速查，不能快速知道谁最久没用；光有链表又不能快速按 key 找节点，所以两者必须配合。

二叉树

题目 推荐解法 最容易错的点
Hot100：二叉树1-二叉树的中序遍历 递归中序遍历 中序顺序别写反，固定是左、根、右。
Hot100：二叉树2-二叉树的最大深度 后序求高度 空树深度是 0，不是 1。
Hot100：二叉树3-翻转二叉树 交换左右子树 交换完左右子树后，再递归处理交换后的左右孩子即可。
Hot100：二叉树4-对称二叉树 镜像比较 对称比较是交叉比较，不是 left.left 对 right.left。
Hot100：二叉树5-二叉树的直径 一边求深度一边更新直径 函数返回的是深度，不是直径；直径要额外用全局变量维护。
Hot100：二叉树6-二叉树的层序遍历 队列按层遍历 每一层开始前要先记录 size，否则同一层和下一层会混在一起。
Hot100：二叉树7-将有序数组转换为二叉搜索树 每次选中点做根节点 边界终止条件是 left > right。
Hot100：二叉树8-验证二叉搜索树 上下界约束 判断范围要用祖先给出的上下界，不是只和父节点比较。
Hot100：二叉树9-二叉搜索树中第 K 小的元素 迭代中序 BST 的第 k 小元素不是随便 DFS，而是依赖中序升序这个性质。
Hot100：二叉树10-二叉树的右视图 每层取最后一个节点 右视图不是简单地一直往右走，因为有些层右孩子可能为空。
Hot100：二叉树11-二叉树展开为链表 栈模拟前序遍历并原地重连 压栈顺序要先右后左，这样弹出时才能保持前序遍历的根、左、右顺序。
Hot100：二叉树12-从前序与中序遍历序列构造二叉树 前序定根，中序分左右 前序数组里左子树的起点紧跟在根节点后面；右子树起点要跨过整棵左子树。
Hot100：二叉树13-路径总和 III 树上前缀和计数 回溯时一定要把当前前缀和次数减回去，否则会污染兄弟分支。
Hot100：二叉树14-二叉树的最近公共祖先 后序判断两侧返回值 递归返回的不一定总是祖先，也可能只是 p 或 q 本身。
Hot100：二叉树15-二叉树中的最大路径和 返回单边最大贡献，顺手更新全局答案 向父节点返回时只能带一条边，不能把左右两边同时返回。

图论

题目 推荐解法 最容易错的点
Hot100：图论1-岛屿数量 遇到陆地就整块淹没 找到新岛屿后，一定要把整块陆地都搜索并标记掉。
Hot100：图论2-腐烂的橘子 多源 BFS 分层扩散 所有初始腐烂橘子都要一起入队，不能只选一个起点。
Hot100：图论3-课程表 入度表 + 队列 建边方向别写反：b -> a 表示先修 b 才能学 a。
Hot100：图论4-实现 Trie (前缀树) 节点保存 26 个孩子 search 和 startsWith 的区别在于：前者必须要求最后节点是单词结尾。

回溯

题目 推荐解法 最容易错的点
Hot100：回溯1-全排列 路径 + used 数组 排列和子集不同，排列要用 used 精确控制元素是否已经被选过。
Hot100：回溯2-子集 从当前位置向后选 子集问题每到一个递归节点，当前路径就是一个答案，不必等到叶子节点。
Hot100：回溯3-电话号码的字母组合 按位扩展路径 空字符串输入时应直接返回空列表，而不是返回包含空串的列表。
Hot100：回溯4-组合总和 同层继续选、下一层还能复用当前元素 允许重复选元素，所以递归下去时传的还是当前下标 i，不是 i + 1。
Hot100：回溯5-括号生成 限制左右括号可用数量 不是“左右括号都能随便放”，而是只有 right > left 时才能放右括号。
Hot100：回溯6-单词搜索 从每个起点做四方向搜索 访问标记必须在回溯时恢复，否则会影响其他起点的搜索。
Hot100：回溯7-分割回文串 切割位置枚举 + 回文判断 不是所有切法都递归，只有当前切出来的前缀是回文时才继续。
Hot100：回溯8-N 皇后 逐行放皇后 主对角线和副对角线都要剪枝，否则搜索量会非常大。

二分查找

题目 推荐解法 最容易错的点
Hot100：二分查找1-搜索插入位置 while (left <= right) 不理解为什么最后返回 left
Hot100：二分查找2-搜索二维矩阵 把矩阵视为一维有序数组 这题和“搜索二维矩阵 II”不同，这里整个矩阵按行展开后是整体有序的。
Hot100：二分查找3-在排序数组中查找元素的第一个和最后一个位置 找左边界 + 找右边界 找左右边界最稳的是两次二分，而不是找到一个位置后再线性扩散。
Hot100：二分查找4-搜索旋转排序数组 判断哪一半有序 关键不是直接判断 target 在左还是右，而是先判断哪一半有序。
Hot100：二分查找5-寻找旋转排序数组中的最小值 和右端点比较 收缩到左半边时，mid 本身不能丢，因为它可能就是最小值。
Hot100：二分查找6-寻找两个正序数组的中位数 划分两个数组 必须在较短数组上二分，这样分割位置 j 才始终合法。

栈

题目 推荐解法 最容易错的点
Hot100：栈1-有效的括号 匹配括号时用后进先出 遇到右括号时要先判断栈是否为空。
Hot100：栈2-最小栈 普通栈 + 最小值栈 最小值栈不是只在出现更小元素时才压栈，而是每次 push 都要同步压入当前最小值。
Hot100：栈3-字符串解码 数字栈 + 字符串栈 数字可能有多位，所以解析重复次数时要写成 num = num * 10 + ...。
Hot100：栈4-每日温度 维护递减温度下标栈 栈里存下标，不存温度值，因为最后要算相隔天数。
Hot100：栈5-柱状图中最大的矩形 弹栈时结算以该柱子为高的最大宽度 想清楚“弹栈时结算”这个时机，比死记公式更重要。

堆

题目 推荐解法 最容易错的点
Hot100：堆1-数组中的第K个最大元素 维护大小为 k 的小根堆 这题保留前 k 大元素时，用的是小根堆而不是大根堆。
Hot100：堆2-前 K 个高频元素 统计频次后用小根堆保留前 K 名 先统计频次，再谈 TopK；不要一边统计一边直接猜排名。
Hot100：堆3-数据流的中位数 大根堆 + 小根堆维护左右两半 两边堆不仅要有序，还要保持大小平衡。

贪心算法

题目 推荐解法 最容易错的点
Hot100：贪心算法1-买卖股票的最佳时机 一边扫一边维护最低买入价 买入必须发生在卖出之前，所以最低价格只能来自当前位置之前。
Hot100：贪心算法2-跳跃游戏 维护当前能到达的最远位置 关键不是选哪一步跳，而是持续维护覆盖范围 farthest。
Hot100：贪心算法3-跳跃游戏 II 按层扩张可达区间 遍历到 nums.length - 2 就够了，因为最后一个位置不需要再起跳。
Hot100：贪心算法4-划分字母区间 用每个字符最后出现位置切段 切段的边界不是看当前字符，而是看这一段所有字符最后出现位置的最远值。

动态规划

题目 推荐解法 最容易错的点
Hot100：动态规划1-爬楼梯 斐波那契型状态转移 初始化要和题意对应，n = 1 时只有一种走法。
Hot100：动态规划2-杨辉三角 当前行由上一行相邻两数推出 每行的首尾固定为 1。
Hot100：动态规划3-打家劫舍 偷或不偷当前房子 别把“能偷更多钱”理解成简单贪心选大值，这题要看相邻约束。
Hot100：动态规划4-完全平方数 完全背包型最少个数 这题求的是最少数量，所以初始值应该设成很大。
Hot100：动态规划5-零钱兑换 完全背包求最少硬币数 初始化极大值时常用 amount + 1，这样不会溢出且足够大。
Hot100：动态规划6-单词拆分 前缀可达性判断 dp[i] 代表的是前 i 个字符能否被拼出，不是以 i 结尾那个字符本身。
Hot100：动态规划7-最长递增子序列 以当前位置结尾的 LIS 这里求的是“子序列”，可以跳过元素，但仍要保持原相对顺序。
Hot100：动态规划8-乘积最大子数组 同时维护最大积和最小积 遇到负数时，最大积和最小积的角色会互换。
Hot100：动态规划9-分割等和子集 0/1 背包装到 target 这题不是完全背包，而是 0/1 背包，所以 j 要倒序遍历。
Hot100：动态规划10-最长有效括号 dp[i] 表示以 i 结尾的最长有效括号长度 dp[i] 不是任意位置都能更新，只有右括号位置才可能形成合法结尾

多维动态规划

题目 推荐解法 最容易错的点
Hot100：多维动态规划1-不同路径 从上和从左转移 第一行和第一列是基础状态，别漏掉初始化。
