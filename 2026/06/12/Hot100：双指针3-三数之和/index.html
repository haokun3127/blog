复习导览

这道题要抓住一句话：先排序，固定一个数，再用左右双指针在后面找另外两个数。

复习点 内容
题型 排序 + 双指针
关键变量 i 固定第一个数，L 左指针，R 右指针
推荐解法 枚举 i，在 i + 1 到末尾之间做两数之和
移动规则 sum < 0 时 L++，sum > 0 时 R--
去重重点 i 要去重，L 和 R 找到答案后也要去重
时间复杂度 O(n^2)
空间复杂度 O(1)，不算返回结果

核心流程

  1. 如果数组为空，或者长度小于 3，直接返回空结果。
  2. 对数组排序。
  3. 从左到右枚举 i，把 nums[i] 固定为三元组的第一个数。
  4. 如果 nums[i] > 0，后面数字都更大，不可能再凑出 0，直接结束。
  5. 如果 nums[i] == nums[i - 1]，说明这个第一个数已经处理过，跳过。
  6. 定义 L = i + 1，R = len - 1。
  7. 计算 sum = nums[i] + nums[L] + nums[R]。
  8. 如果 sum == 0，加入答案，并移动 L 和 R 跳过重复值。
  9. 如果 sum < 0，说明当前和太小，L++。
  10. 如果 sum > 0，说明当前和太大，R--。

题目 核心思路 指针含义 是否排序
两数之和 哈希表找补数 没有双指针 不需要

面试表达

如果面试官问：三数之和怎么做？

可以这样回答：

我会先对数组排序，然后枚举第一个数 nums[i]。固定 i 后，在 i + 1 到数组末尾之间用左右双指针找另外两个数。如果三数之和小于 0，说明需要更大的数，左指针右移；如果三数之和大于 0，说明需要更小的数，右指针左移；如果等于 0，就把三元组加入结果。为了避免重复答案，固定 i 时要跳过和前一个相同的值，找到答案后也要跳过 L 和 R 的重复值。排序是 O(n log n)，双指针主流程是 O(n^2)，所以总时间复杂度是 O(n^2)。

如果继续问：为什么 i 要去重？

可以这样回答：

因为排序后相同数字会挨在一起。如果当前 nums[i] 和 nums[i - 1] 相同，说明上一次已经用同样的值作为第一个数找过所有组合了，这次再固定它会产生重复三元组，所以直接跳过。
