Hot100：双指针1-移动零

发表于2026-06-10|更新于2026-06-12|算法

题目

给定一个数组 nums，将所有 0 移动到数组末尾，同时保持非零元素的相对顺序。

要求：

  • 必须在原数组上操作，不能拷贝额外数组。
  • 尽量减少操作次数。

示例：

复习导览

这道题要抓住一句话：先把非零元素按原顺序放到前面，再把剩余位置补 0。

这里的关键变量是 j：

复习点 内容
题型 双指针 / 原地数组
关键变量 j，下一个非零元素写入位置
推荐解法 两次遍历：先搬非零，再补零
是否保持相对顺序 保持
时间复杂度 O(n)
空间复杂度 O(1)
最容易错的点 忘记最后补 0，或者把非零元素顺序打乱

破题思路

数组里真正需要保留顺序的是非零元素。

例如：

移动零之后，数组前半部分应该是非零元素：

剩下的位置全部补成 0：

所以这道题不需要真的“把 0 一个一个往后挪”，更自然的思路是：

推荐解法：双指针写入法

核心流程

  1. 定义 j = 0，表示下一个非零元素应该写入的位置。
  2. 从左到右遍历数组。
  3. 如果 nums[i] != 0，说明当前元素要保留，把它写到 nums[j]。
  4. 写入后 j++，准备接收下一个非零元素。
  5. 第一轮结束后，[0, j - 1] 都是非零元素。
  6. 从 j 到末尾全部填 0。

代码逐行理解

这是防御性判断。如果传进来的数组是 null，直接返回，避免后面访问 nums.length 时出现空指针异常。

LeetCode 原题通常不会传 null，但平时自己写方法时加上这个判断也可以。

j 是这道题最重要的指针。

它不是普通遍历指针，而是“写入指针”：

开始时还没有放任何非零元素，所以第一个非零元素应该放在下标 0。

i 是遍历指针，用来从左到右扫描数组中的每个元素。

可以理解为：

只有当前元素不是 0 时，才需要把它保留下来。

遇到 0 时什么都不做，因为 0 最后会统一补到后面。

这两行表示：

因为 i 是从左到右扫描，所以非零元素写入的顺序也一定是原来的顺序。这样就满足题目要求：保持非零元素相对顺序不变。

第一轮结束后，数组前面已经放好了全部非零元素。

从 j 开始到数组末尾，这些位置都应该是 0，所以统一补零。

样例手推

以 nums = [0, 1, 0, 3, 12] 为例：

步骤 i nums[i] j 操作 数组状态
初始 - - 0 - [0,1,0,3,12]
1 0 0 0 遇到 0，跳过 [0,1,0,3,12]
2 1 1 0 nums[0] = 1，j++ [1,1,0,3,12]
3 2 0 1 遇到 0，跳过 [1,1,0,3,12]
4 3 3 1 nums[1] = 3，j++ [1,3,0,3,12]
5 4 12 2 nums[2] = 12，j++ [1,3,12,3,12]
补零 3-4 - 3 后面补 0 [1,3,12,0,0]

中间出现 [1,3,12,3,12] 不用担心，因为后半部分属于“待覆盖区域”，最后会被统一改成 0。

为什么这个方法能保持顺序

因为第一轮遍历时，i 从左到右扫描。

每次遇到非零元素，就按扫描顺序写入 j：

所以非零元素之间的相对顺序不会变。

这也是这题和普通交换题的区别：

和交换写法的区别

这题还有一种常见写法：遇到非零元素时，和 j 位置交换。

交换写法也能通过，但这篇笔记中的“先写非零，再补零”更好理解。

两种写法对比：

写法 思路 特点
写入 + 补零 先收集非零元素，再补零 逻辑清楚，适合理解
交换法 非零元素和前面的 0 交换 一次遍历，但交换次数可能更多

如果面试时写这题，推荐先写“写入 + 补零”版本。它更容易说明正确性，也不容易把顺序搞乱。

复杂度分析

时间复杂度：O(n)。

第一轮遍历数组一次，第二轮最多再遍历数组一次，所以整体仍然是线性时间。

空间复杂度：O(1)。

只用了常数个变量 i、j，没有创建额外数组。

易错点

易错点 说明
忘记补 0 只把非零元素前移，后面会残留旧值
把 j++ 放错位置 只有写入非零元素后才移动 j
误以为中间数组状态必须正确 第一轮结束前后半部分可以是旧值，最后会补零
破坏相对顺序 不能随便把后面的非零元素换到前面
多开一个数组 题目要求原地修改

面试表达

如果面试官问：移动零这题怎么做？

可以这样回答：

我会用双指针原地处理。i 负责从左到右扫描数组，j 表示下一个非零元素应该写入的位置。遇到非零元素时，把它写到 nums[j]，然后 j++。第一轮结束后，数组前 j 个位置就是所有非零元素，并且顺序和原数组一致。最后从 j 到末尾全部补成 0。时间复杂度是 O(n)，空间复杂度是 O(1)。

如果面试官问：为什么非零元素顺序不会乱？

可以这样回答：

因为遍历指针 i 是从左到右扫描的，遇到非零元素后也是按这个顺序依次写入 j 指向的位置，所以先出现的非零元素一定先写入，后出现的非零元素一定后写入，相对顺序自然保持不变。
