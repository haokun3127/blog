Hot100：矩阵3-旋转图像

发表于2026-07-08|更新于2026-07-08|算法

题目

给定一个 n x n 的二维矩阵 matrix 表示一个图像，请你将图像顺时针旋转 90 度。要求必须在原地旋转。

参考入口：

  • 力扣题目：48. 旋转图像

复习导览

这道题要抓住一句话：顺时针旋转 90 度，等价于先沿主对角线转置，再把每一行左右翻转。

复习点 内容
题型 矩阵变换
推荐解法 转置 + 每行反转
关键变量 i、j、n
时间复杂度 O(n^2)
空间复杂度 O(1)
最容易错的点 转置时内层循环必须从 j = i + 1 开始，否则会把元素交换两次。

破题思路

直接按旋转后的坐标关系写原地赋值容易绕晕，而且临时变量很多。更稳的做法是把旋转拆成两个熟悉的矩阵操作：先转置，让行列交换；再反转每一行，让方向转成顺时针。

核心流程

  1. 先遍历主对角线右上方区域，交换 matrix[i][j] 和 matrix[j][i]。
  2. 转置完成后，每一行都做一次左右反转。
  3. 左右反转可以用双指针首尾交换实现。
  4. 完成这两步后，矩阵就已经顺时针旋转 90 度。

代码

复杂度分析

  • 时间复杂度：O(n^2)。
  • 空间复杂度：O(1)。

易错点

  • 转置时内层循环必须从 j = i + 1 开始，否则会把元素交换两次。
  • 这题要求原地修改矩阵，不能额外开一个新矩阵来接收结果。

一句话总结

旋转图像的核心不是死记代码，而是先抓住“顺时针旋转 90 度，等价于先沿主对角线转置，再把每一行左右翻转。”这条主线，再把模板写稳。
