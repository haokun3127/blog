学习导览

前面已经学过 MCP 的概念、Tools、Resources、Prompts。这一篇开始从 Java 工程视角看 MCP：官方 Java SDK 并不是一个“自动帮你做智能体”的框架，而是一套把 MCP 协议落到 Java 代码里的基础设施。

可以把它理解成四层：

也就是说，SDK 解决的是“协议怎么稳定通信”的问题，不直接解决“模型怎么判断该调哪个工具”的问题。

知识速览

模块 作用 复习重点
mcp-core MCP 核心协议实现 Schema、Session、Client、Server
mcp-json-jackson2 Jackson 2 序列化支持 兼容旧项目
mcp-json-jackson3 Jackson 3 序列化支持 适合新版本生态

Sync API 同步调用 简单、易理解，适合普通业务
Async API 异步调用 适合高并发和响应式链路

详细笔记

modelcontextprotocol 是官方大本营

MCP 官方生态里有规范文档，也有不同语言的 SDK。Java SDK 的定位是让 Java 项目能够实现 MCP Client 或 MCP Server。

这里要区分两件事：

所以学习 SDK 时，不能只看怎么写工具注册代码，还要理解它背后是在处理 JSON-RPC 消息、连接生命周期和能力声明。

Schema 层：协议的字典

McpSchema 可以理解成 MCP 协议在 Java 里的“字典”。协议中出现的工具、资源、Prompt、初始化请求、工具调用请求、工具调用响应，都会被映射成 Java 数据结构。

常见对象可以这样记：

类型 对应场景 说明
Tool 工具定义 包含工具名、描述、参数 Schema
CallToolRequest tools/call 请求 Client 请求 Server 执行某个工具
CallToolResult 工具执行结果 Server 返回执行结果
Resource 资源定义 包含 URI、名称、描述、MIME 类型
Prompt Prompt 模板定义 包含模板名称、描述和参数
InitializeRequest 初始化请求 建立连接时进行能力协商
InitializeResult 初始化响应 返回 Server 支持的能力

学习 MCP 时要注意：模型看到的“工具”只是上层表现，底层真正传输的是标准化的 JSON-RPC 消息。

Transport 层：消息怎么送

Transport 只负责“送消息”，不负责理解消息。它不关心这是工具调用、资源读取还是 Prompt 获取，只负责把 JSON-RPC 从一端传到另一端。

Stdio Transport

Stdio 使用标准输入和标准输出通信。MCP Server 作为一个本地子进程启动，Client 往它的 stdin 写 JSON-RPC 请求，再从 stdout 读取响应。

这种方式适合本地工具，比如 Claude Desktop 启动本地 Java MCP Server。优点是简单、没有网络依赖；缺点是只适合本地，不适合企业级远程部署。

Streamable HTTP

Streamable HTTP 是更适合远程 MCP 服务的方式。它让请求和响应都可以通过 HTTP 流式传输，不再依赖额外的 SSE 长连接。

Session 层：维护一次 MCP 会话

Transport 只负责传输，但一次 MCP 通信还需要管理状态。Session 主要负责三件事：

  1. 连接初始化：通过 initialize 交换能力声明。
  2. 请求响应配对：多个请求并发时，靠 JSON-RPC 的 id 找到对应响应。
  3. 通知处理：处理工具列表变更、资源变更等不需要响应的消息。

如果没有 Session 层，Client 发出多个请求后就很难知道哪个响应对应哪个请求。

Client/Server 层：开发者直接使用的 API

SDK 最上层提供 McpServer 和 McpClient。开发者一般通过 Builder 模式配置服务信息、注册工具、注册资源、注册 Prompt，再启动 MCP Server。

同步 API 更适合普通项目，写起来直观；异步 API 更适合高并发或响应式场景，但理解成本更高。

官方 SDK 和 Spring AI 的关系

官方 SDK 更底层，负责协议实现；Spring AI MCP 集成更偏框架封装，负责和 Spring Boot 的自动配置、注解、Bean 生命周期结合。

可以这样理解：

如果要理解 MCP 的底层通信，应该看官方 SDK；如果要快速在 Spring Boot 项目里暴露工具，Spring AI 会更方便。

面试或项目讲解时可以这样说：

MCP 官方 Java SDK 本质上是 MCP 协议在 Java 里的实现。它把 JSON-RPC 消息抽象成 Schema，把 stdin/stdout、SSE、Streamable HTTP 抽象成 Transport，再通过 Session 管理初始化、能力协商和请求响应配对，最后提供 Client/Server API 给业务系统注册工具、资源和 Prompt。
