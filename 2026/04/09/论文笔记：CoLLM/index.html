论文标题 CoLLM: Integrating Collaborative Embeddings into Large Language Models for Recommendation
期刊/会议 原笔记未注明，仅记录发表日期为 2025-06-14
发表日期 2025-06-14
作者 Yang Zhang；Fuli Feng；Jizhi Zhang；Keqin Bao；Qifan Wang；Xiangnan He
核心主题 将传统协同信息以独立模态形式注入 LLM 推荐框架
关键词 CoLLM、LLMRec、Collaborative Information、Warm-start、Cold-start
原文 PDF Zhang 等 - 2025 - CoLLM Integrating Collaborative Embeddings into Large Language Models for Recommendation.pdf
笔记创建日期 2026/04/09

摘要速览

CoLLM 聚焦于一个很关键但常被忽略的问题：现有 LLM 推荐方法更擅长利用文本语义处理冷启动，却没有充分吸收传统推荐系统中的协同信息，因此在热启动场景下常常不如矩阵分解或图模型。本文把协同 embedding 视为一种独立模态，通过映射层对齐到 LLM 的 token 空间，在不修改 LLM 主体结构的前提下，让大模型同时具备文本推理能力与协同建模优势。实验表明，CoLLM 在 ML-1M 和 Amazon-Book 上都优于传统模型和 TALLRec，并在冷、热场景之间取得了更好的平衡。

项目 内容
英文摘要 Leveraging Large Language Models as recommenders, referred to as LLMRec, is gaining traction and brings novel dynamics for modeling user preferences, particularly for cold-start users. However, existing LLMRec approaches primarily focus on text semantics and overlook the crucial aspect of incorporating collaborative information from user-item interactions, leading to potentially sub-optimal performance in warm-start scenarios. To ensure superior recommendations across both warm and cold scenarios, we introduce CoLLM, an innovative LLMRec approach that explicitly integrates collaborative information for recommendations. CoLLM treats collaborative information as a distinct modality, directly encoding it from well-established traditional collaborative models, and then tunes a mapping module to align this collaborative information with the LLM’s input text token space for recommendations. By externally integrating traditional models, CoLLM ensures effective collaborative information modeling without modifying the LLM itself, providing the flexibility to adopt diverse collaborative information modeling mechanisms. Extensive experimentation validates that CoLLM adeptly integrates collaborative information into LLMs, resulting in enhanced recommendation performance.
中文摘要翻译 现有 LLMRec 方法在冷启动场景中能借助文本语义取得不错效果，但往往忽略用户与物品交互中蕴含的协同信息，因此在热启动场景容易出现性能瓶颈。CoLLM 将协同信息视作独立模态，直接从传统协同模型中提取表示，再通过映射模块将其对齐到 LLM 输入空间，从而在不改动 LLM 主体的前提下完成融合。实验结果表明，该方法能够有效提升推荐性能，并兼顾冷启动与热启动场景。

1. 文章基本框架

背景

如果说 TALLRec 解决了 LLM “能不能做推荐”，那么 CoLLM 进一步追问的是：LLM 能不能在保住语义泛化优势的同时，补齐传统推荐系统最强的协同建模能力。

  • LLMRec 的偏科问题：现有 LLM 推荐方法依赖文本提示和语义推理，在冷启动用户或物品上往往表现不错，但这种优势并不能直接迁移到热启动场景。
  • 协同信息缺失：传统推荐模型的强项在于从大规模交互数据中挖掘协同信号，而把用户和物品完全转成文本后，这部分信息会被显著削弱。
  • 热启动表现不足：当用户和物品已经拥有丰富历史交互时，纯文本驱动的 LLMRec 反而可能落后于 MF、LightGCN 等经典方法。

目的

本文的核心目标是把“协同信息”作为一种可以被 LLM 理解的新模态，让推荐模型在冷启动和热启动场景下都保持竞争力。

  • 显式注入协同信号：不再只依赖文本描述，而是把传统推荐模型学到的 user/item embedding 直接接入 LLM。
  • 保持模型轻量与灵活：作者不修改 LLM 主干结构，也不扩充庞大的 ID 词表，而是通过一个小型映射层完成跨模态对齐。
  • 兼顾泛化与效果：既要保留 LLM 的语义理解优势，也要吸收传统协同模型在热启动场景中的稳定收益。

结论

CoLLM 证明了“LLM + 外挂协同信息”是一条务实且有效的推荐路线。

  • 提出了一套两阶段微调框架，把 MF、LightGCN、DIN 等传统协同模型与 Vicuna-7B 结合起来。
  • 在整体指标上，CoLLM 超越了传统 SOTA 和纯 LLMRec 方法，同时显著缓解了 LLMRec 在热启动场景中的短板。

2. 结果与讨论

数据以及数据来源

  • 数据集：使用 ML-1M 和 Amazon-Book 两个经典推荐数据集。
  • 数据划分：采用严格的时间戳切分，避免信息泄露，并将任务处理为隐式反馈二分类问题。
  • 冷热拆分：测试集进一步划分为 Warm 与 Cold 子集，用来检验模型在不同数据稠密度场景下的表现差异。

方法

CoLLM 的思路很直接：让传统推荐模型负责提取协同表示，让 LLM 负责理解文本上下文并做最终推理。

1. 协同信息编码模块（CIE: Collaborative Information Encoding）

  • **协同模型 f_psi**：使用 MF、LightGCN 或 DIN 等传统模型，把用户和物品编码成低维协同 embedding。
  • **映射层 g_phi**：通过一个 MLP 将协同 embedding 映射到 LLM 的词向量空间，充当“翻译器”。
  • 混合输入方式：最终输入被组织为文本 token 与协同向量交织的序列，使 LLM 能在统一空间中处理两类信息。

2. 两阶段解耦微调（Two-Step Tuning）

  • Step 1: 文本能力与任务对齐

    • 暂时屏蔽协同 embedding，只用文本提示微调 LoRA。
    • 目标是先让 LLM 理解推荐任务本身，并稳住冷启动场景中的文本推理能力。

  • Step 2: 协同信号对齐

    • 冻结 LLM 与 LoRA，仅更新 CIE 中的映射层，必要时可联合微调协同模型。
    • 目标是把传统协同信号对齐到 LLM 已经适配好的表示空间中。

结论分析

  • 整体性能（RQ1）：无论底层协同模型选用 MF、LightGCN 还是 DIN，CoLLM 都优于纯传统模型和 TALLRec，说明“语义 + 协同”组合确实有效。
  • 冷热场景对比：在 Warm 子集上，CoLLM 明显修复了 TALLRec 的短板；在 Cold 子集上，又基本保留了 TALLRec 的优势。
  • 消融与效率（RQ2）：去掉 CIE 后性能显著下降，而直接引入 ID token 的朴素做法甚至会带来负收益。两阶段训练带来的额外开销相对可控，文中报告训练时间约增加 15.5%，推理耗时约增加 12.5%。

3. 文章亮点思考

1. 本文有什么优缺点？

  • 优点

    • 问题抓得准：文章非常敏锐地指出了 LLMRec 在热启动场景中的核心弱点，不是简单追求“大模型更强”，而是回到推荐系统最本质的协同信号上。
    • 设计足够克制：作者没有硬改 LLM 结构，而是用轻量映射层做对齐，这让方案兼具研究价值和工程可落地性。
    • 可插拔性强：CIE 模块能兼容多种传统推荐模型，说明 CoLLM 更像一个融合框架，而不是只绑定某一种底座。

  • 缺点 / 疑问

    • 在线推理成本仍高：即使加入协同信息后效果提升明显，LLM 级别的推理延迟在真实工业重排场景里仍然是压力点。
    • 两阶段训练可能次优：Step 2 只对齐映射层，没有和 LoRA 共同端到端联合优化，可能限制最终上限。

2. 可以如何改进？

  1. 端到端联合优化：让文本表示、协同表示和映射层在统一目标下协同训练，减少阶段间割裂。
  2. 动态门控机制：针对极端冷启动样本，引入 gating 模块动态决定当前应更依赖文本语义还是协同信号。
  3. 面向部署的轻量化设计：可以进一步探索蒸馏、缓存协同表示或只在精排阶段调用 LLM，以降低线上时延。

3. 这篇文章为什么重要？

这篇文章的重要性在于，它把 LLM 推荐研究从“把推荐问题翻译成文本”推进到了“如何把非文本协同知识高效接入 LLM”。CoLLM 没有否定传统推荐模型，而是把它们当作成熟的外部专家来协同工作。这种思路既尊重了推荐系统长期积累的方法论，也为后续多模态推荐、结构化信号接入大模型提供了非常自然的范式。
