在 GitHub 创建远程仓库

在 GitHub 上创建一个新仓库：

  1. 打开 GitHub；
  2. 点击右上角 +；
  3. 选择 New repository；
  4. 输入仓库名，例如 chatbot-api；
  5. 建议不要勾选 README、.gitignore、License，避免和本地项目冲突；
  6. 创建完成后，复制 GitHub 提供的仓库地址。

Push 时要求登录 GitHub

如果使用 HTTPS，GitHub 可能会弹出浏览器让你登录。

如果要求输入密码，需要注意：GitHub 现在不支持直接使用账号密码推送代码，需要使用下面几种方式之一：

  • GitHub 浏览器授权；
  • Personal Access Token；
  • SSH Key。

新手建议优先使用浏览器授权。

Push 前安全检查

每次 push 前建议检查：

重点确认不要提交以下内容：

  • .env
  • 密码
  • Token
  • API Key
  • 数据库账号和密码
  • 本地 IDE 临时文件
  • 编译产物，例如 target/

如果不确定某个文件能不能提交，先不要 push，检查清楚后再提交。

  • 用 git remote add origin 仓库地址 绑定 GitHub 仓库；

熟悉这套流程后，就可以比较顺畅地把本地代码管理起来，并同步到 GitHub。
