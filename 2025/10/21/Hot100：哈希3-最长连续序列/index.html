问题描述

给定一个未排序的整数数组 nums，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 O(n) 的算法解决此问题。

示例 2：

提示：

  • 0 <= nums.length <= 10^5
  • -10^9 <= nums[i] <= 10^9

核心思想

算法思路

关键洞察： 每个数都判断一次这个数是不是连续序列的开头那个数。

判断方法： 用哈希表查找这个数前面一个数是否存在，即 num-1 在序列中是否存在。存在那这个数肯定不是开头，直接跳过。

时间复杂度： 因此只需要对每个开头的数进行循环，直到这个序列不再连续，因此复杂度是 O(n)。

算法步骤

  1. 去重存储：将所有数字存入 HashSet，自动去重并提供 O(1) 查找
  2. 寻找序列起点：对每个数字，检查 num-1 是否存在
  3. 扩展序列：从起点开始，依次查找 num+1, num+2, ... 直到不连续
  4. 更新最大长度：记录当前序列长度，更新全局最大值

复杂度分析

时间复杂度：O(n)

关键分析： 虽然有嵌套循环，但每个元素最多被访问常数次。

详细说明：

  • 外层循环：遍历 HashSet 中的每个元素，O(n)
  • 内层 while 循环：看似会增加复杂度，但实际上：
    • 每个数字最多作为序列起点被扩展一次
    • 每个数字最多在扩展过程中被访问一次
    • 总的扩展次数不会超过 n

举例说明：
对于序列 [1,2,3,4,5]：

  • 只有 1 会作为起点进行扩展
  • 2,3,4,5 都会被跳过（因为它们的前驱存在）
  • 扩展过程访问 1→2→3→4→5，总共 5 次
  • 总时间复杂度仍然是 O(n)

空间复杂度：O(n)

  • HashSet 存储所有元素：O(n)
  • 其他变量：O(1)
  • 总空间复杂度：O(n)

算法优化思路

为什么不用排序？

排序方法：

HashSet 方法的优势：

  • 时间复杂度更优：O(n) vs O(n log n)
  • 不需要修改原数组
  • 逻辑更清晰

为什么使用 HashSet 而不是其他数据结构？

数据结构 查找时间 插入时间 空间复杂度 适用性
HashSet O(1) O(1) O(n) ✅ 最优
TreeSet O(log n) O(log n) O(n) ❌ 查找慢
ArrayList O(n) O(1) O(n) ❌ 查找慢
数组 O(n) O(1) O(n) ❌ 查找慢

关键优化点

  1. 避免重复计算

  2. 利用 HashSet 的 O(1) 查找

学习总结

算法思想

  1. 哈希表优化查找

    • 将 O(n) 的查找优化为 O(1)
    • 是解决查找问题的常用技巧

  2. 避免重复计算

    • 通过判断序列起点，避免重复扩展
    • 将看似 O(n²) 的算法优化为 O(n)

  3. 问题转化

    • 将”找最长连续序列”转化为”找序列起点并扩展”
    • 化繁为简的思维方式

实战技巧

  1. 数据结构选择

    • 需要频繁查找 → HashSet
    • 需要有序 → TreeSet
    • 需要索引 → ArrayList

  2. 时间复杂度分析

    • 嵌套循环不一定是 O(n²)
    • 要分析每个元素被访问的总次数

  3. 代码优化

    • 先实现正确的算法
    • 再分析瓶颈进行优化
    • HashSet 是优化查找的利器

易错点

  1. 忘记去重

  2. 边界条件处理

  3. 变量命名混淆

总结

最长连续序列问题是一道经典的哈希表应用题，通过这道题我们可以学习到：

  • 哈希表在优化查找中的重要作用
  • 如何通过巧妙的算法设计避免重复计算
  • 时间复杂度分析的技巧
  • 数据结构选择的重要性

关键要点：

  • 使用 HashSet 实现 O(1) 查找
  • 只对序列起点进行扩展，避免重复计算
  • 总时间复杂度 O(n)，满足题目要求

推荐做法：

  • 优先使用 HashSet 方法（时间复杂度最优）
  • 理解算法的核心思想：寻找序列起点
  • 掌握 HashSet 的基本用法和特性
