两种初始化方式的区别

静态初始化

  • 特点：手动指定数组的元素，系统会根据元素的个数自动计算出数组的长度
  • 适用场景：已经明确了要操作的所有数据

动态初始化

  • 特点：手动指定数组长度，由系统给出默认初始化值
  • 适用场景：只明确元素个数，但是不明确具体的数据

💡 选择建议：

  • 如果已知具体数据 → 使用静态初始化
  • 如果只知道数量 → 使用动态初始化

注意事项2：void方法的return

void 表示无返回值，可以省略 return，也可以单独书写 return（后面不加数据）

⚠️ 注意：

  • void 方法不能 return 数据
  • return 语句后面不能有其他代码

组成部分 说明

定义方法的两个明确

定义方法时，要做到两个明确：

  1. 明确返回值类型

    • 如果方法没有返回数据 → 写 void
    • 如果方法有返回数据 → 写对应的数据类型

  2. 明确参数

    • 明确参数的类型
    • 明确参数的数量

调用方法的注意事项

  • void 类型的方法 → 直接调用
  • 非 void 类型的方法 → 推荐用变量接收返回值

方法重载（Method Overloading）

什么是方法重载？

方法重载指在同一个类中定义多个同名但参数不同的方法。

方法重载的条件

方法之间构成重载必须同时满足以下条件：

  1. ✅ 多个方法在同一个类中
  2. ✅ 多个方法具有相同的方法名
  3. ✅ 多个方法的参数不同（类型不同 或 数量不同）

⚠️ 重要：

  • 重载与返回值类型无关
  • 重载与方法调用无关，只看方法定义
