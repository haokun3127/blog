Hot100：哈希2.字母异位词分组

问题描述

给你一个字符串数组，请你将 字母异位词 组合在一起。可以按任意顺序返回结果列表。

字母异位词 是由重新排列源单词的所有字母得到的一个新单词。

提示：

  • 1 <= strs.length <= 10^4
  • 0 <= strs[i].length <= 100
  • strs[i] 仅包含小写字母

核心概念

什么是字母异位词？

字母异位词（Anagram）：两个字符串互为字母异位词，当且仅当两个字符串包含的字母相同。

关键特性：

  • 包含相同的字母
  • 每个字母出现的次数相同
  • 字母顺序可以不同

例如：

  • "eat", "tea", "ate" 是一组字母异位词
  • "tan", "nat" 是一组字母异位词
  • "bat" 单独一组

解法：排序 + 哈希表

思路分析

同一组字母异位词中的字符串具备相同点，可以使用相同点作为一组字母异位词的标志。

核心思想：

  1. 由于互为字母异位词的字符串包含的字母相同
  2. 对字符串进行排序后得到的结果一定相同
  3. 将排序后的字符串作为哈希表的键（key）
  4. 将原始字符串加入对应键的列表中（value）

算法步骤：

  1. 创建哈希表，键为排序后的字符串，值为字母异位词列表
  2. 遍历每个字符串
  3. 将字符串转为字符数组并排序
  4. 使用排序后的字符串作为键
  5. 将原始字符串加入对应的列表
  6. 返回哈希表中所有的值

复杂度分析

时间复杂度：O(n × k × log k)

  • n 是字符串数组的长度
  • k 是字符串的最大长度
  • 需要遍历 n 个字符串
  • 对每个字符串，排序需要 O(k log k) 时间
  • 哈希表操作需要 O(1) 时间
  • 总时间：O(n × k × log k)

空间复杂度：O(n × k)

  • n 是字符串数组的长度
  • k 是字符串的最大长度
  • 哈希表需要存储所有字符串
  • 最坏情况下每个字符串都不是异位词，需要 n 个键值对
  • 每个字符串平均长度为 k

解法二：计数法 + 哈希表

思路分析

虽然排序法已经很高效，但我们还可以通过计数的方式来避免排序操作。

核心思想：

  1. 统计每个字符串中每个字母出现的次数
  2. 将字母和出现次数按顺序拼接成字符串作为键
  3. 相同的字母异位词会产生相同的计数键

算法步骤：

  1. 创建哈希表，键为计数字符串，值为字母异位词列表
  2. 遍历每个字符串
  3. 统计每个字母（a-z）出现的次数
  4. 将字母和次数拼接成字符串作为键（如 “a1e1t1”）
  5. 将原始字符串加入对应的列表
  6. 返回哈希表中所有的值

为什么使用 HashMap？

核心优势：

  1. 快速分组 - O(1) 时间查找和插入
  2. 自动去重 - 相同的 key 自动归为一组
  3. 灵活的值类型 - 可以存储列表

学习总结

算法思想

  1. 寻找共同特征

    • 字母异位词的共同特征是：排序后结果相同
    • 利用这个特征作为分组的依据

  2. 哈希表分组

    • 使用哈希表高效地将具有相同特征的元素分组
    • key 存储特征，value 存储具有该特征的元素列表

  3. 时间空间权衡

    • 使用 O(n × k) 的空间换取 O(n × k log k) 的时间
    • 相比暴力法 O(n² × k) 有显著提升

实战技巧

  1. 字符串处理

    • 熟练掌握字符串和字符数组的转换
    • 理解排序的作用和时间复杂度

  2. HashMap 的高级用法

    • getOrDefault() 简化代码
    • map.values() 获取所有值

  3. 代码优化

    • 选择合适的数据结构（HashMap）
    • 找到问题的关键特征（排序后相同）

易错点

  1. 忘记将列表放回 map

  2. 字符数组转字符串的方式

  3. 返回值类型

总结

字母异位词分组是一道典型的哈希表应用题，通过这道题我们可以学习到：

  • 问题抽象 - 将字母异位词问题转化为寻找共同特征的问题
  • 哈希表分组 - 利用哈希表高效地对元素进行分组
  • 字符串处理 - 字符串、字符数组的转换和排序
  • 时间优化 - 从 O(n²) 优化到 O(n × k log k)

关键要点：

  • 排序后的字符串可以作为字母异位词的唯一标识
  • HashMap 是分组问题的利器
  • getOrDefault() 可以简化代码逻辑
  • 注意字符串和字符数组的正确转换

推荐做法：

  • 先掌握排序法（代码简洁）
  • 再学习计数法（时间更优）
  • 理解哈希表在分组问题中的应用
