Hot100：哈希2-字母异位词分组

发表于2025-10-20|更新于2026-06-03|算法

题目

给定一个字符串数组 strs，将字母异位词组合在一起。可以按任意顺序返回结果。

字母异位词指的是：由相同字母重新排列得到的字符串，每个字母出现次数也必须相同。

示例：

提示：

  • 1 <= strs.length <= 10^4
  • 0 <= strs[i].length <= 100
  • strs[i] 只包含小写字母

复习导览

这道题要抓住一句话：一组字母异位词必须拥有同一个分组标识。

比如：

它们排序后的结果一样，所以可以用 "aet" 作为这一组的 key。

复习点 内容
题型 哈希表分组
关键动作 给每个字符串生成统一 key
推荐解法 排序字符串作为 key
优化解法 统计 26 个字母次数作为 key
时间复杂度 排序法 O(n * k log k)，计数法 O(n * k)
最容易错的点 char[] 不能直接用 toString() 当 key

其中 n 是字符串数量，k 是字符串最大长度。

破题思路

要把异位词放到同一组，关键是找到它们的共同特征。

异位词的共同特征有两个：

  • 字母种类相同。
  • 每个字母出现次数相同。

所以有两种常见 key：

key 设计 示例 特点
排序后的字符串 "eat" -> "aet" 代码简单，容易写对
字母次数编码 "eat" -> a1e1t1 不需要排序，时间更优

复习时建议先牢牢记住排序法，因为它最直观；再理解计数法为什么更快。

解法一：排序 + HashMap

这道题第一眼看上去，很多人会下意识去想：要不要拿一个字符串和后面的字符串逐个比较？

其实不用。

这道题更好的思路是：先给每个字符串生成一个统一的分组标识 key，再用 HashMap 按 key 分组。

对于字母异位词来说，虽然原字符串顺序不同，但它们包含的字母和每个字母出现的次数完全一样，所以只要把字符串排个序，它们就会变成同一个结果。

例如：

于是我们就可以把排序后的 "aet" 当作这一组的 key。

核心流程

  1. 创建 Map<String, List<String>>。
  2. 遍历每个字符串 str。
  3. 把 str 转成字符数组并排序。
  4. 用排序后的字符串作为 key。
  5. 把原字符串加入这个 key 对应的列表。
  6. 最后返回 map.values()。

手推过程

以 ["eat","tea","tan","ate","nat","bat"] 为例：

原字符串 排序后 key 加入分组后 map
"eat" "aet" aet -> [eat]
"tea" "aet" aet -> [eat, tea]
"tan" "ant" ant -> [tan]
"ate" "aet" aet -> [eat, tea, ate]
"nat" "ant" ant -> [tan, nat]
"bat" "abt" abt -> [bat]

可以按代码执行过程再口头顺一遍：

  • 处理 eat，排序后得到 aet，于是放进 map["aet"]。
  • 处理 tea，排序后还是 aet，于是继续放进 map["aet"]。
  • 处理 tan，排序后得到 ant，于是放进 map["ant"]。
  • 处理 ate，排序后还是 aet，继续加入 map["aet"]。
  • 处理 nat，排序后还是 ant，继续加入 map["ant"]。
  • 处理 bat，排序后得到 abt，放进 map["abt"]。

先把每个字符串排序，得到统一 key；再用 HashMap 按 key 分组。

解法二：计数 + HashMap

排序法每个字符串都要排序，时间是 O(k log k)。由于题目说字符串只包含小写字母，可以用长度为 26 的数组统计字母次数。

两种解法对比

对比项 排序法 计数法
key 排序后的字符串 字母出现次数编码
时间复杂度 O(n * k log k) O(n * k)
空间复杂度 O(n * k) O(n * k)
优点 直观，容易写 避免排序，更快
注意点 new String(arr) key 必须有分隔或字母标记

为什么计数法 key 需要字母标记？

如果只拼数字，可能出现歧义。例如不同计数组合都可能拼成相似字符串。加上字母后，a1b11 和 a11b1 就能区分开。

复杂度分析

排序法：

  • 时间复杂度：O(n * k log k)。每个字符串排序需要 O(k log k)。
  • 空间复杂度：O(n * k)。哈希表中要存储所有字符串分组。

计数法：

  • 时间复杂度：O(n * (k + 26))，通常写作 O(n * k)。
  • 空间复杂度：O(n * k)。额外的 count[26] 是常数空间。

2. 忘记把字符串放进对应列表

如果少了 list.add(str)，当前字符串就不会真正进入分组。

如果少了 map.put(key, list)，那么当 key 第一次出现时，新建的空列表就不会被放回哈希表里。

补充一下：很多题解会把这三步压缩成一行 computeIfAbsent(...)，本质上做的事情是一样的，只是写法更简洁。

3. 认为输出顺序必须和示例一样

题目说可以按任意顺序返回，所以分组顺序、组内顺序通常不要求完全一致。

4. 没有注意字符范围

计数法这里能用 int[26]，是因为题目限制了只包含小写字母。如果有大写字母、数字或 Unicode 字符，就不能直接套这个版本。

复盘清单

复习这道题时，重点确认自己能回答这些问题：

  • 为什么排序后的字符串可以作为同组 key？
  • HashMap 的 key 和 value 分别是什么？
  • getOrDefault(key, new ArrayList<>()) 这行代码在做什么？
  • 为什么 list.add(str) 之后还要 map.put(key, list)？
  • 为什么 char[] 要用 new String(arr)？
  • 计数法为什么能把时间复杂度降到 O(n * k)？
  • 如果字符串里有大写字母，计数法要怎么调整？

一句话总结

字母异位词分组的核心是“给同类字符串生成同一个 key”。排序法用排序结果做 key，计数法用字母次数做 key，本质都是用哈希表完成分组。
