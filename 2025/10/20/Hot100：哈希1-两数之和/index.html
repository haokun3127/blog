Hot100：哈希1-两数之和

发表于2025-10-20|更新于2026-05-30|算法
|浏览量:

题目

给定一个整数数组 nums 和一个整数目标值 target，在数组中找出和为 target 的两个整数，并返回它们的数组下标。

要求：

  • 每种输入只会对应一个答案。
  • 同一个元素不能重复使用。
  • 返回下标顺序不限。

示例：

复习导览

这道题要抓住一句话：遍历当前数 x 时，只需要快速判断前面是否出现过 target - x。

暴力法慢，是因为查找另一个数需要再扫一遍数组；哈希表法快，是因为把已经遍历过的数放进 HashMap，之后查找补数可以接近 O(1)。

复习点 内容
题型 哈希表查找
关键变量 target - nums[i]
推荐解法 一次遍历 + HashMap
时间复杂度 O(n)
空间复杂度 O(n)
最容易错的点 先查再放，避免同一个元素用两次

推荐解法：HashMap 一次遍历

核心流程

  1. 创建一个 HashMap<Integer, Integer>，保存已经遍历过的数字和下标。
  2. 从左到右遍历数组。
  3. 对当前数字 nums[i]，计算补数 need = target - nums[i]。
  4. 如果 map 里有 need，说明已经找到答案，返回 [map.get(need), i]。
  5. 如果没有找到，再把当前数字和下标放入 map。

注意这里必须是先查找，再插入当前元素。

如果先插入再查找，在 target = 6、当前数字是 3 的时候，就可能让当前元素和自己匹配。

代码逐行理解

保存已经看过的数字。比如遍历完 2 后，map 里就是 {2=0}。

当前数字还缺什么数字才能凑成 target。

如果之前已经出现过补数，就可以直接返回之前补数的下标和当前下标。

当前数暂时没找到搭档，就把它存起来，给后面的数字使用。

手推过程

以 nums = [2, 7, 11, 15]，target = 9 为例：

i nums[i] need = target - nums[i] map 中是否有 need 操作 map
0 2 7 没有 存入 2 -> 0 {2=0}
1 7 2 有 返回 [0, 1] {2=0}

再看一个有重复数字的例子：

i nums[i] need map 操作
0 3 3 {} 找不到，存入 3 -> 0
1 3 3 {3=0} 找到，返回 [0, 1]

这说明“先查再放”既能处理重复元素，又不会把同一个元素用两次。

暴力解法对比

暴力法是双层循环，枚举所有两个数的组合：

暴力法也能做出来，但它每次都要继续往后找另一个数，时间复杂度是 O(n^2)。哈希表法相当于提前建立了“数字到下标”的查找表，把找补数这件事变快。

复杂度分析

HashMap 解法：

  • 时间复杂度：O(n)。数组只遍历一次，containsKey 和 get 平均是 O(1)。
  • 空间复杂度：O(n)。最坏情况下，要把大部分数字存进哈希表。

暴力解法：

  • 时间复杂度：O(n^2)。
  • 空间复杂度：O(1)。

易错点

1. 把下标和值搞反

本题要求返回的是下标，不是数字本身。

2. 先 put 再 containsKey

这样可能会让当前元素匹配自己。

3. 处理重复元素时想复杂了

不用额外处理重复元素。只要先查再放，[3, 3] 这种情况自然能返回两个不同下标。

4. 忘记题目保证只有一个答案

题目说只有一个答案，所以找到后可以直接 return，不需要继续找所有组合。

复盘清单

复习这道题时，重点确认自己能回答这几个问题：

  • HashMap 的 key 和 value 分别存什么？
  • 为什么要计算 target - nums[i]？
  • 为什么必须先查找再插入？
  • 遇到 [3, 3]、target = 6 时，代码怎么执行？
  • 为什么哈希表能把时间复杂度从 O(n^2) 降到 O(n)？

一句话总结

两数之和的核心不是“凑两个数”，而是把问题转成“当前数需要的补数，之前有没有出现过”。HashMap 用空间换时间，把补数查找从线性查找优化成常数级查找。
