Hot100：哈希1.两数之和

发表于2025-10-20|更新于2025-10-30|算法
|浏览量:

问题描述

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出和为目标值 target 的那两个整数，并返回它们的数组下标。

要求：

  • 你可以假设每种输入只会对应一个答案
  • 但是，数组中同一个元素在答案里不能重复出现
  • 你可以按任意顺序返回答案

示例：

解法一：哈希表（推荐）

思路分析

使用哈希表可以将寻找 target - x 的时间复杂度从 O(N) 降低到 O(1)，这是最优的解法。

核心思想：

  1. 创建一个哈希表用于存储已遍历过的元素及其索引
  2. 对于每一个元素 x，首先查询哈希表中是否存在 target - x
  3. 如果存在，说明找到了答案，返回两个索引
  4. 如果不存在，将 x 和它的索引插入到哈希表中
  5. 这样可以保证不会让元素和自己匹配

代码实现

执行过程示例

以 nums = [2, 7, 11, 15], target = 9 为例：

步骤 i nums[i] target - nums[i] map 中是否存在? map 内容 操作
1 0 2 7 ❌ {2: 0} 将 (2, 0) 存入 map
2 1 7 2 ✅ {2: 0} 找到答案！返回 [0, 1]

复杂度分析

  • 时间复杂度：O(N)

    • 其中 N 是数组中的元素数量
    • 对于每一个元素 x，我们可以 O(1) 地寻找 target - x
    • 只需要遍历一次数组

  • 空间复杂度：O(N)

    • 其中 N 是数组中的元素数量
    • 主要为哈希表的开销
    • 最坏情况下需要存储 N-1 个元素

解法二：暴力枚举

思路分析

最容易想到的方法是枚举数组中的每一个数 x，然后遍历整个数组寻找是否存在 target - x。

核心思想：

  1. 使用双层循环遍历所有可能的两数组合
  2. 外层循环遍历第一个数 nums[i]
  3. 内层循环从 i + 1 开始遍历第二个数 nums[j]
  4. 判断 nums[i] + nums[j] 是否等于 target
  5. 如果相等，返回索引 [i, j]

优化点：

  • 内层循环从 i + 1 开始，避免重复匹配
  • 每个元素不能被使用两次

代码实现

执行过程示例

以 nums = [2, 7, 11, 15], target = 13 为例：

i j nums[i] nums[j] nums[i] + nums[j] 是否等于 target?
0 1 2 7 9 ❌
0 2 2 11 13 ✅ 返回 [0, 2]

复杂度分析

  • 时间复杂度：O(N²)

    • 其中 N 是数组中的元素数量
    • 最坏情况下数组中任意两个数都要被匹配一次
    • 外层循环 N 次，内层循环平均 N/2 次

  • 空间复杂度：O(1)

    • 只使用了常数级别的额外空间

性能对比

时间复杂度 O(N) O(N²)
空间复杂度 O(N) O(1)
适用场景 大数据量 小数据量
实现难度 中等 简单

优缺点分析

哈希表法：

  • ✅ 时间效率高，适合大规模数据
  • ✅ 只需遍历一次数组
  • ❌ 需要额外的空间存储哈希表
  • ❌ 实现相对复杂

暴力枚举法：

  • ✅ 实现简单，易于理解
  • ✅ 不需要额外空间
  • ❌ 时间复杂度高，数据量大时性能差
  • ❌ 需要嵌套循环

3. 算法优化的核心原理

  • 传统查找: 在数组中线性查找 target - x 需要 O(N) 时间
  • 哈希表查找: 在哈希表中查找 target - x 只需要 O(1) 时间

复杂度深入分析

时间复杂度详细对比：

  • 哈希表方法: O(N)

    • 单次遍历数组：N 次迭代
    • 每次哈希查找：O(1) 平均时间
    • 总时间：N × O(1) = O(N)

  • 暴力枚举: O(N²)

    • 外层循环：N 次迭代
    • 内层循环：平均 N/2 次迭代
    • 总时间：N × N/2 = O(N²)

空间复杂度详细对比：

  • 哈希表方法: O(N) - 最坏情况下存储 N-1 个元素
  • 暴力枚举: O(1) - 只使用几个变量的额外空间

实际性能差异：

  • N = 100: 哈希表 100 次操作 vs 暴力枚举 5,000 次操作
  • N = 1,000: 哈希表 1,000 次操作 vs 暴力枚举 500,000 次操作
  • N = 10,000: 哈希表 10,000 次操作 vs 暴力枚举 50,000,000 次操作

学习总结

算法思想

  1. 以空间换时间

    • 哈希表法通过使用额外的空间（O(N)）来换取时间效率的提升（从 O(N²) 到 O(N)）
    • 这是算法优化中的常见策略

  2. 查找优化

    • 将数组的线性查找（O(N)）优化为哈希表的常数时间查找（O(1)）
    • HashMap 是解决查找问题的利器

  3. 避免重复计算

    • 暴力法中通过 j = i + 1 避免重复匹配
    • 哈希表法通过先查找再插入避免自身匹配

总结

两数之和问题是学习算法的经典入门题目，通过这道题我们学到了：

  1. 算法优化的核心思想：以空间换时间，将 O(N²) 优化到 O(N)
  2. 数据结构的重要性：HashMap 在解决查找问题中的强大作用
  3. 代码实现的细节：异常处理、边界条件、注释规范等
  4. 复杂度分析：理论分析与实际性能的对应关系

这道题为后续学习更复杂的算法和数据结构打下了坚实的基础。建议多动手实践，深入理解每一个实现细节。
