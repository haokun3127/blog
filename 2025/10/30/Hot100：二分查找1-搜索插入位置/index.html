题目描述

LeetCode 35. 搜索插入位置

给定一个排序数组和一个目标值，在数组中找到目标值，并返回其索引。如果目标值不存在于数组中，返回它将会被按顺序插入的位置。

请必须使用时间复杂度为 O(log n) 的算法。

问题理解

这道题有两个核心任务：

  1. 查找：如果目标值存在，返回其索引
  2. 定位插入位置：如果目标值不存在，找到它应该插入的位置（保持数组有序）

解题思路

算法选择

由于题目要求时间复杂度为 O(log n)，并且数组是有序的，这是使用二分查找算法的典型场景。

核心思路

使用二分查找在有序数组中定位目标值或插入位置：

  1. 初始化搜索区间：min = 0，max = nums.length - 1
  2. 循环查找：
    • 计算中间位置：mid = (min + max) / 2
    • 比较 nums[mid] 与 target：
      • nums[mid] == target：✅ 找到目标，返回 mid
      • nums[mid] > target：目标在左半部分，max = mid - 1
      • nums[mid] < target：目标在右半部分，min = mid + 1
  3. 退出条件：min > max 时，目标值不存在

🔑 关键理解：插入位置的确定

这是本题的核心！当二分查找结束（min > max）时，为什么 min 就是插入位置？

数学证明

循环不变式：在每次循环中，min 左侧的所有元素都 < target，max 右侧的所有元素都 > target

当循环结束时（min > max）：

  • min 指向第一个 ≥ target 的位置
  • 在 min 位置插入 target 可以保持数组有序

三种情况分析

  1. 目标值小于所有元素：min = 0，插入到开头
  2. 目标值大于所有元素：min = nums.length，插入到末尾
  3. 目标值在某两个元素之间：min 指向第一个大于目标值的位置

算法分析

时间复杂度

O(log n)

每次循环都会将搜索区间缩小一半，因此最多需要 log₂n 次比较。

空间复杂度

O(1)

只使用了常数个额外变量（min、max、mid），不随输入规模增长。

输入：nums = [1, 3, 5, 6], target = 5

步骤 min max mid nums[mid] 判断 操作 说明
1 0 3 1 3 3 < 5 min = 2 目标在右半部分
2 2 3 2 5 5 == 5 返回 2 找到目标

结果：返回 2，目标值存在于索引 2

示例 3：插入到末尾

输入：nums = [1, 3, 5, 6], target = 7

步骤 min max mid nums[mid] 判断 操作 说明
1 0 3 1 3 3 < 7 min = 2 目标在右半部分
2 2 3 2 5 5 < 7 min = 3 目标在右半部分
3 3 3 3 6 6 < 7 min = 4 目标在右半部分
4 4 3 - - min > max 返回 4 插入到末尾

结果：返回 4，表示 7 应该插入到索引 4（数组末尾）
插入后：[1, 3, 5, 6, 7] ✅ 保持有序

💡 核心要点

  1. 二分查找的本质：通过不断缩小搜索区间，将时间复杂度降低到 O(log n)

  3. 循环不变式：维护 min 和 max 的正确含义是关键
